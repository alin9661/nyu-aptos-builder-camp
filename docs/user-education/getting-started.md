# Getting Started with NYU Nexus

Welcome to NYU Nexus! This guide will help you get started with your Aptos wallet and the governance platform.

## Your First Steps

### 1. Access NYU Nexus

**Login**:
1. Visit [nexus.nyu.edu](https://nexus.nyu.edu)
2. Click "Sign In with NYU"
3. Enter your NetID and password
4. Complete two-factor authentication (if enabled)
5. You'll be redirected to your dashboard

**First Time Login**:
- A wallet is automatically created for you
- You'll see a welcome modal with your wallet information
- Your wallet address is displayed prominently
- Review the educational materials provided

### 2. View Your Wallet

**Navigate to Wallet**:
1. Click "Wallet" in the sidebar
2. View your wallet address and balance
3. See recent transactions
4. Copy your address to share with others

**Wallet Information**:
- **Address**: Your unique identifier on Aptos blockchain
- **Balance**: Current APT token balance
- **Network**: Testnet or Mainnet
- **Transaction History**: All your past transactions

### 3. Explore the Dashboard

**Key Sections**:
- **Overview**: Platform statistics and recent activity
- **Proposals**: Active governance proposals
- **Reimbursements**: Reimbursement requests
- **Treasury**: Organization funds and balances
- **Elections**: Upcoming and active elections

### 4. Check Your Balance

**Testnet Funding**:
If on testnet, your wallet may be automatically funded with test APT tokens.

**View Balance**:
- Check wallet page for current balance
- Balance shown in APT tokens
- Equivalent USD value may be displayed
- Transaction history shows all changes

**Getting More APT** (Testnet):
- Contact administrator for additional testnet tokens
- Use Aptos faucet (if available)
- Note: Testnet tokens have no real-world value

## How to Submit a Reimbursement Request

### Step 1: Prepare Documentation

**Required Information**:
- Expense description
- Amount (in USD or APT)
- Receipt/proof of purchase
- Category (travel, supplies, etc.)
- Date of expense

**Supporting Documents**:
- Upload receipt images (JPG, PNG, PDF)
- Include detailed description
- Specify payment method preference

### Step 2: Create Request

1. Navigate to "Reimbursements" page
2. Click "Submit New Request"
3. Fill out the form:
   - **Title**: Brief description
   - **Amount**: Cost of expense
   - **Description**: Detailed explanation
   - **Category**: Select appropriate category
   - **Attachments**: Upload receipts
4. Review all information carefully
5. Click "Submit Request"

### Step 3: Transaction Signing

1. Review transaction details
2. Confirm amount and recipient
3. Click "Sign Transaction"
4. Wait for blockchain confirmation
5. Receive confirmation notification

### Step 4: Track Status

**Request States**:
- **Pending**: Awaiting treasurer review
- **Approved**: Ready for disbursement
- **Rejected**: Not approved (see reason)
- **Paid**: Funds sent to your wallet

**Notifications**:
- You'll receive email and in-app notifications
- Status updates appear in notification center
- Check reimbursements page for details

## How to Participate in Governance

### View Active Proposals

**Finding Proposals**:
1. Go to "Governance" in sidebar
2. View list of active proposals
3. Click proposal to see details
4. Read description and discussion

**Proposal Information**:
- **Title**: Brief summary
- **Description**: Full details
- **Proposer**: Who created it
- **Voting Period**: When voting ends
- **Current Results**: Live vote counts

### Cast Your Vote

**Voting Process**:
1. Open a proposal
2. Read the full description
3. Review discussion and comments
4. Decide your position (For/Against/Abstain)
5. Click your choice
6. Confirm transaction
7. Sign with your wallet
8. Wait for blockchain confirmation

**Voting Power**:
- Typically 1 vote per member
- May vary based on governance rules
- Check your voting weight in settings
- Votes are permanent once cast

**Vote Visibility**:
- All votes are public on blockchain
- Transparency ensures accountability
- Vote history is permanent
- Your vote cannot be changed

### Delegate Your Votes (If Available)

**Delegation**:
Some governance systems allow delegation:
1. Choose a trusted delegate
2. Transfer voting power temporarily
3. Revoke delegation anytime
4. Monitor delegate's voting record

## How to Request Funding

### Treasury Requests

**When to Request**:
- Event funding
- Project expenses
- Supply purchases
- Service contracts

**Request Process**:
1. Navigate to "Treasury"
2. Click "Request Funding"
3. Complete form:
   - **Purpose**: Why you need funds
   - **Amount**: How much you need
   - **Budget Breakdown**: Itemized costs
   - **Timeline**: When funds are needed
   - **Impact**: Expected outcomes
4. Submit for review
5. Await treasurer/board approval

### Approval Process

**Review Steps**:
1. Treasurer reviews request
2. Board/committee may vote
3. Approval or rejection decision
4. Funds disbursed if approved

**Timeline**:
- Small requests: 1-3 days
- Large requests: 1-2 weeks
- Emergency requests: Same day (if available)

## Understanding Notifications

### Notification Types

**Wallet Notifications**:
- Wallet created
- Funds received
- Transaction confirmed
- Low balance warning

**Governance Notifications**:
- New proposal created
- Voting period ending soon
- Your vote recorded
- Proposal result finalized

**Reimbursement Notifications**:
- Request submitted
- Request approved/rejected
- Payment processed
- Additional information needed

**Security Notifications**:
- Login from new device
- Password change
- Unusual activity detected
- Security updates

### Notification Settings

**Customize Notifications**:
1. Go to Settings
2. Click "Notifications"
3. Toggle preferences:
   - Email notifications
   - In-app notifications
   - Security alerts
   - Educational content
4. Save changes

**Unsubscribe**:
- Click unsubscribe link in emails
- Or update preferences in settings
- Critical security alerts cannot be disabled

## Common Tasks

### Check Transaction Status

**Using NYU Nexus**:
1. Go to Wallet page
2. Click "Transactions" tab
3. Find your transaction
4. View status and details

**Using Aptos Explorer**:
1. Copy transaction hash
2. Visit [explorer.aptoslabs.com](https://explorer.aptoslabs.com)
3. Paste hash in search
4. View detailed transaction information

### Update Your Profile

**Profile Settings**:
1. Click your name/avatar
2. Select "Settings"
3. Update:
   - Display name
   - Email
   - Notification preferences
   - Language (if available)
4. Save changes

### View Transaction History

**Full History**:
1. Go to Wallet page
2. Click "Transaction History"
3. Filter by:
   - Date range
   - Transaction type
   - Status
4. Export to CSV (if available)

### Export Data

**Available Exports**:
- Transaction history (CSV)
- Reimbursement history
- Voting record
- Account activity log

**How to Export**:
1. Navigate to relevant section
2. Click "Export" button
3. Select format (CSV, PDF)
4. Download file

### Troubleshooting

**Can't Log In?**
- Verify NetID and password
- Check two-factor authentication
- Contact NYU IT if account locked
- Clear browser cache and try again

**Transaction Failed?**
- Check balance for gas fees
- Verify network connection
- Try again after a few minutes
- Contact support if persists

**Not Receiving Notifications?**
- Check spam/junk folder
- Verify email address in settings
- Ensure notifications enabled
- Check browser notification permissions

## Best Practices

### Security
- Never share your NYU password
- Enable two-factor authentication
- Log out after each session
- Use secure networks only

### Transactions
- Always verify recipient addresses
- Double-check amounts before confirming
- Save transaction hashes for records
- Review transaction details carefully

### Governance
- Read proposals thoroughly
- Participate in discussions
- Vote on time before deadline
- Stay informed about decisions

### Communication
- Use official channels only
- Verify sender identity
- Report suspicious messages
- Keep contact information updated
